# Recommendation Framework

## Structure
Every recommendation must include:
- `id`: Unique identifier.
- `priority`: 1 (Highest) to N.
- `description`: Actionable step.
- `expectedImprovement`: Quantifiable metric.
- `estimatedCost`: Time, resources, or financial cost.
- `implementationDifficulty`: Low, Medium, High.

## Generation
- Generated by `RecommendationEngine` analyzing the optimal `SimulationScenario`.
